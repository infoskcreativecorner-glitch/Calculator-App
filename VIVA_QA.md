# Viva Questions and Answers - Calculator Application

## 1. What is the main objective of your project?
The objective is to build a browser-based calculator that performs basic arithmetic operations with a clean UI and proper error handling.

## 2. Which technologies did you use and why?
I used HTML for structure, CSS for design and layout, and JavaScript for logic and interactivity because these are core web technologies and easy to deploy.

## 3. What operations are supported in your calculator?
The calculator supports addition, subtraction, multiplication, and division, along with parentheses and decimal values.

## 4. How does your calculator evaluate expressions?
It stores user input as an expression string, validates allowed characters, and evaluates the expression using JavaScript logic.

## 5. How did you handle invalid expressions?
I used validation checks and try-catch blocks. If an expression is invalid, the app shows an error message and resets safely.

## 6. How do you handle division by zero?
After evaluation, I check whether the result is finite. If not, I show an error message instead of displaying Infinity.

## 7. Why did you separate HTML, CSS, and JavaScript files?
Separation improves code readability, maintainability, and follows standard software development practice.

## 8. What is DOM in JavaScript?
DOM stands for Document Object Model. It represents HTML elements as objects so JavaScript can dynamically read and modify them.

## 9. What event listeners are used in your project?
I used click event listeners for calculator buttons and keydown listeners for keyboard input support.

## 10. What is the purpose of the Clear button?
The Clear button resets the entire expression, removes error messages, and sets the display back to zero.

## 11. What is the purpose of Backspace?
Backspace removes the last entered character, allowing users to quickly correct typing mistakes.

## 12. How is responsive design achieved?
Responsive design is done using flexible layout, CSS grid, and media queries so the calculator works on both desktop and mobile screens.

## 13. What are the limitations of your project?
It does not currently include scientific functions, calculation history, or user personalization options.

## 14. How can this project be improved in future?
Future enhancements include scientific functions, theme switcher, history storage, and PWA support.

## 15. What testing did you perform?
I performed functional testing for all operations, invalid input testing, keyboard testing, and UI testing for responsiveness.

## 16. What is the difference between `==` and `===` in JavaScript?
`==` checks value with type conversion, while `===` checks both value and type strictly.

## 17. Why is error handling important in this project?
It prevents crashes, improves user experience, and ensures reliable output even for unexpected input.

## 18. What is a function in JavaScript?
A function is a reusable block of code that performs a specific task when called.

## 19. Why did you choose a web-based calculator instead of desktop software?
Web apps are easy to run, require no installation, and work across operating systems through a browser.

## 20. What did you learn from this project?
I learned UI design, event-driven programming, input validation, exception handling, and how to document a complete software project professionally.
